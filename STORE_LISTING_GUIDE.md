# Chrome Web Store 上架指南

## 1. 准备工作

### 1.1 开发者账号注册

1. 访问 [Chrome Web Store 开发者控制台](https://chrome.google.com/webstore/developer/dashboard)
2. 使用Google账号登录
3. 支付一次性注册费用（目前为5美元）

### 1.2 上传材料准备

- **扩展包**：doubao-waterprint-downloader-v1.0.zip
- **图标**：
  - 128x128像素（必需）
  - 48x48像素（建议）
  - 16x16像素（建议）
- **截图**：至少1张，最多5张（1280x800或640x400）
- **隐私政策链接**：指向您的隐私政策页面
- **详细描述**：扩展功能和用途说明

## 2. 上架步骤

### 2.1 创建新项目

1. 在开发者控制台中点击"添加新项目"
2. 上传扩展包（doubao-waterprint-downloader-v1.0.zip）
3. 填写基本信息：
   - **名称**：豆包无水印图片提取工具
   - **描述**：提取豆包网站生成的无水印图片，支持批量下载
   - **类别**：工具 > 生产力
   - **语言**：简体中文

### 2.2 填写详细描述

```
豆包无水印图片提取工具是一款专门为豆包网站用户设计的浏览器扩展，帮助您轻松获取高质量无水印图片。

主要功能：
✅ 自动检测豆包生成的图片
✅ 提取并下载无水印高质量图片
✅ 提供友好的图片预览界面
✅ 显示每张图片的生成提示词
✅ 支持批量选择和下载

使用方法：
1. 访问豆包网站并生成图片
2. 点击页面顶部的"下载"按钮
3. 在弹窗中浏览并选择要下载的图片
4. 点击下载按钮保存到本地

技术特点：
- 完全本地处理，不上传任何数据
- 实时监听API响应，自动更新图片列表
- 支持Chrome、Edge等Chromium内核浏览器

本工具仅供学习和个人使用，请遵守豆包网站的使用条款和版权规定。
```

### 2.3 隐私政策

1. 在"隐私政策实践"部分添加您的隐私政策链接
2. 如果您有网站，可以将PRIVACY_POLICY.md上传到网站并提供链接
3. 如果没有网站，可以使用GitHub Pages或创建一个简单的静态网站

### 2.4 权限说明

在"权限"部分，为每个请求的权限提供明确的用途说明：

- **activeTab**：与当前活动标签页交互，检测豆包网站
- **storage**：在您的设备上临时存储最近的图片信息
- **downloads**：将您选择的无水印图片保存到您的设备
- **host_permissions**：访问豆包网站和图片域名，不访问其他网站
- **scripting**：将必要脚本注入到豆包网站，实现图片提取功能
- **notifications**：显示下载成功或失败的通知
- **webRequest**：拦截豆包网站的API请求，不会查看其他网站请求

### 2.5 可发布性检查

Chrome Web Store会对您的扩展进行自动检查，确保：
1. 无恶意代码
2. 权限使用合理
3. 隐私政策完整
4. 遵循Chrome扩展开发最佳实践

## 3. 审核与发布

### 3.1 审核流程

1. 提交后，您的扩展将进入审核队列
2. 审核时间通常为3-7个工作日
3. 审核员可能会要求您提供更多信息或修改内容
4. 审核通过后，您可以选择立即发布或安排未来发布

### 3.2 常见被拒原因

1. **权限过多**：确保只请求必要的权限
2. **隐私政策不明确**：详细说明数据收集和使用情况
3. **功能描述不准确**：确保描述与实际功能一致
4. **违反Chrome政策**：查看[Chrome Web Store政策](https://developer.chrome.com/docs/webstore/program-policies/)

### 3.3 提高通过率的技巧

1. **代码注释**：在关键代码部分添加注释，解释其用途
2. **最小权限原则**：确保每个权限都有明确的用途
3. **用户控制**：确保用户可以完全控制扩展的行为
4. **透明度**：在描述中清楚地说明扩展的工作原理

## 4. 发布后管理

### 4.1 用户反馈

- 积极回应用户评论和问题
- 根据用户反馈持续改进功能
- 定期更新以适配豆包网站的变化

### 4.2 版本更新

1. 修改代码后，更新manifest.json中的版本号
2. 重新打包并上传新版本
3. 在发布说明中详细说明更新内容
4. 注意：新版本需要重新审核

### 4.3 数据分析

Chrome Web Store提供以下数据：
- 下载量
- 用户评价
- 使用统计
- 错误报告

定期查看这些数据，了解用户需求和使用情况。

## 5. 其他浏览器市场

### 5.1 Microsoft Edge Add-ons

1. 访问 [Microsoft Edge加载项开发人员仪表板](https://partner.microsoft.com/dashboard/microsoftedge)
2. 使用相同的扩展包（基于Chromium的扩展基本兼容）
3. 填写类似的描述和权限说明

### 5.2 Firefox Add-ons

1. Firefox使用不同的扩展系统(WebExtensions)，可能需要修改manifest.json
2. 访问 [Firefox Add-on Developer Hub](https://addons.mozilla.org/developers/)
3. 注意Firefox的审核政策可能与Chrome有所不同

## 6. 法律与合规

### 6.1 知识产权

- 确保使用的图标、名称和描述不侵犯他人版权
- 考虑为自己的扩展申请商标
- 在必要时添加适当的免责声明

### 6.2 数据保护

- 即使您的扩展不收集数据，也应提供隐私政策
- 遵守GDPR、CCPA等数据保护法规
- 考虑用户权利，如访问、删除等

### 6.3 内容责任

- 明确扩展的用途和限制
- 添加适当的免责声明
- 考虑年龄分级（如果适用）

---

**上架扩展是一个需要耐心和细致的过程。确保所有信息准确、完整，并遵循平台指南，将大大提高通过审核的可能性。**
